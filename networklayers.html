Layers of OSI Model

The 7 layers of Network are also called as 'OSI' Model, Which means 'Open System Interconnection'. It was developed by the organization called 'ISO' (International organization for Standardization) in the year of 1984.

This is seven layer architecture with each layer having specific functionality to perform. All the 7 layers work together to transmit the data from one person to another across the world.

  1. 1. Physical Layer (Layer 1):

    This layer is the lowest layer of OSI model. It is the actual Physical Connection between the devices. In this layer the information is in the form of bits. It transfers the bits from one node to the other node. After receiving the data it will convert the data into 0s and 1s and send them to the next layer which is Data Link Layer.

  2. 2. Data Link Layer (Layer 2):

    The data link layer is responsible for the node to node delivery of the message. Its aim is to transfer the data which is error-free from one node to another through the Physical layer.If a packet arrives in a network, Data Link Layer is responsible to transfer it to the MAC addressThe packet received from the network layer is further divided into frames.

  3. 3. Network Layer (Layer 3):

    This Layer will work on the Transmission of data from one host to other who are located in differernt networks. It takes care of somthing called packet routing which means, the selection of the shortest path to transmit the packet, from the routes which are available.

  4. 4.Transport Layer (Layer 4):

    This layer take the services from the network layer and provide it to the application later. The data in this layer is referred as Segments. This takes the responsibility of End to End Delivery of the complete message. It will also checks if the data is transfered successfully. If not, it will re-transmit the data.

  5. 5. Session Layer (Layer 5):

    Session Layer is responsible for the establisment of the connection, Maintenance of Sessions, Authentication and also make sure to give Security.This Layer allows the two proccesses to establish, use and terminate a connection.This will avoid the data by using a method called Synchronization. It allows the systems to communicate.

  6. 6. Presentation Layer (Layer 6):

    This Layer is also called as Translation Layer. The data will be translated to the required format to transfer over the network. This will also do the Encryption and Decryption to ensure the security. And also compress the bits by reducing the length of the bits by not loosing the data.

  7. 7. Application Layer (Layer 7):

    This is the top layer in the OSI Model. This is a level of Application where we access the information by displaying the data.
